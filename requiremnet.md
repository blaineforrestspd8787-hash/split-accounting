# 产品需求文档（PRD）：自由职业者分账助手

| 文档版本 | 修改日期   | 修改人 | 修改内容       |
| -------- | ---------- | ------ | -------------- |

## 1. 产品概述

### 1.1 产品背景
许多自由职业者（如设计师、开发人员、咨询顾问等）每日收入不固定，且需要按固定比例（如30%）向介绍人或合伙人分成。目前用户多采用微信记录、Excel表格或脑力记忆，存在**漏记、算错、月底统计痛苦**等痛点，缺乏一款简单、直观、高频使用的记账工具。

### 1.2 产品定位
**自由职业者的分账助手**——专注于“收入记录-自动分成-到手利润”的极简记账工具。非通用记账软件，而是为特定人群（有固定分账比例的自由职业者）提供“每天10秒完成记账”的核心价值。

### 1.3 核心价值主张
- **随手记**：每日打开网页，输入总收入，自动算出利润和分成。
- **不纠结**：无需自己计算百分比，无需月底汇总。
- **看得清**：本月/今年累计利润（实际到手金额）一目了然。
- **可导出**：一键导出CSV，用于对账或财务归档。

### 1.4 目标用户
- **自由职业者/接单者**：每天收入波动大，需要向合伙人/介绍人支付固定比例分成。
- **使用特征**：对复杂财务软件排斥，偏好低学习成本、高频率使用的工具。
- **典型场景**：每天工作结束后，记录当天收入，快速知道自己实际赚了多少以及合伙人应得多少。

## 2. 产品范围与优先级

### 2.1 MVP范围（P0 - 必须实现）
| 模块         | 功能点                                     |
| ------------ | ------------------------------------------ |
| 每日收入录入 | 默认当天日期，输入总收入，实时计算利润与分成 |
| 修改当天记录 | 若当天已有记录，自动变更为“更新”模式         |
| 本月统计     | 显示本月总收入、总利润、总分成               |
| 今年统计     | 显示今年总收入、总利润、总分成               |
| 历史记录查看 | 按月份筛选，展示每日明细                     |
| 编辑历史记录 | 支持修改任意日期的总收入，自动重算并更新统计 |
| CSV导出      | 导出当前筛选月份的数据为CSV文件              |

### 2.2 后续迭代（P1）
- 自定义分成比例（默认30%，可改为20%、40%等）
- 多合伙人分账（记录多个分成对象）
- 收入趋势图表（折线图展示每日/每月收入变化）
- 手机端适配优化（响应式设计）
- 月度利润目标功能（设定目标，展示完成进度）
- 数据云同步（简单登录系统，跨设备使用）

### 2.3 暂不考虑（P2）
- 完整财务系统（发票、税务、多项目核算）
- 团队协作/消息功能
- 预算管理或账单支付功能

## 3. 功能需求详细说明

### 3.1 首页（核心使用页）

**页面布局建议**：
```
+--------------------------------------------------+
|  今日记账                              [日期选择器] |
|  今日总收入：  [  输入框   ]  元                   |
|  ┌─────────────┐  ┌─────────────┐               |
|  │ 今日利润(到手) │  │ 合伙人分成   │               |
|  │   ¥ 0.00    │  │   ¥ 0.00    │               |
|  │   (70%)     │  │   (30%)     │               |
|  └─────────────┘  └─────────────┘               |
|                     [ 保存 ] [ 更新（如已存在） ]   |
+--------------------------------------------------+
|  本月统计（6月）                                 |
|  ┌─────────┐ ┌─────────┐ ┌─────────┐          |
|  │总收入    │ │总利润    │ │总分成    │          |
|  │¥12,000  │ │¥8,400   │ │¥3,600   │          |
|  └─────────┘ └─────────┘ └─────────┘          |
+--------------------------------------------------+
|  今年统计（2026）                               |
|  （同上三个卡片）                               |
+--------------------------------------------------+
|  最近记录（最近7天）                             |
|  日期   总收入   利润   分成   操作              |
|  06-11  ¥1200  ¥840  ¥360  编辑               |
|  ...                                            |
+--------------------------------------------------+
```

**交互细节**：
- **日期选择器**：默认为“今天”，用户可切换其他日期（用于补录历史记录）。
- **实时计算**：用户在“今日总收入”输入框中输入数字时，下方利润和分成卡片数值动态变化（使用 JavaScript 监听 input 事件）。
- **保存/更新逻辑**：
  - 如果当前日期无记录：显示“保存”按钮，点击后新增记录。
  - 如果当前日期已有记录：显示“更新”按钮，点击后覆盖原有总收入并重新计算利润和分成。
- **数据验证**：总收入必须为非负数字，最多两位小数（自动格式化）。
- **最近记录**：展示最近7天（或最近10条）记录，每条后面带“编辑”按钮，点击后将日期和总收入填入顶部表单，方便快速修改。

### 3.2 历史记录页

**页面功能**：
- **月份筛选器**：下拉选择年份+月份（如“2026年6月”），默认显示当前月份。
- **数据表格**：
  | 日期 | 总收入 | 利润(70%) | 分成(30%) | 操作 |
  |---|---|---|---|---|
  | 2026-06-11 | ¥1,200 | ¥840 | ¥360 | 编辑 删除 |
- **编辑**：点击编辑，弹出模态框或跳转至首页（自动填入日期和原收入），修改后保存并刷新表格。
- **删除**：二次确认后删除记录，并重新计算该月份统计。
- **导出CSV**：按钮位于表格上方，导出当前筛选月份的所有记录（字段：日期、总收入、利润、分成），文件名格式“收入记录_2026年6月.csv”。

### 3.3 设置页（P0无，P1补充说明）

用于后续版本允许用户修改分成比例（默认30%）。建议在顶部导航增加“设置”入口，存储用户自定义比例到 localStorage。

## 4. 数据模型

### 4.1 前端存储方案（MVP采用 localStorage）

| 键名               | 值类型                        | 说明                       |
| ------------------ | ----------------------------- | -------------------------- |
| `incomeRecords`    | Array<IncomeRecord>           | 存储所有收入记录             |
| `splitPercentage`  | Number (可选，P1)              | 分成比例，默认30，单位%      |

**IncomeRecord 对象结构**：
```javascript
{
  id: "uuid或者Date.now()",      // 唯一标识
  date: "2026-06-11",            // 字符串 YYYY-MM-DD
  totalIncome: 1200.00,          // 数字，单位元
  profit: 840.00,                // 可选，可不存实时计算
  partnerShare: 360.00,          // 可选，可不存实时计算
  createdAt: "2026-06-11T10:00:00Z",
  updatedAt: "2026-06-11T10:00:00Z"
}
```

**读取与写入**：
- 加载页面时从 `localStorage` 读取 `incomeRecords`，若无则初始化空数组。
- 每次新增/更新/删除操作后，更新 `localStorage` 并刷新页面统计与表格。

### 4.2 数据计算规则
- **利润 = 总收入 × (1 - 分成比例)**，默认分成比例30%，即利润 = 总收入 × 70%。
- **分成 = 总收入 × 分成比例**，默认总收入 × 30%。
- 所有金额保留两位小数，四舍五入。

## 5. 非功能需求

### 5.1 性能要求
- 页面首屏加载时间 < 1秒（localStorage 读写极快）。
- 支持存储至少10年的每日记录（3650条），表格渲染不卡顿。

### 5.2 兼容性
- 现代浏览器（Chrome, Edge, Safari, Firefox 最近两个主要版本）。
- 移动端浏览器基本可用（响应式布局，但P1再做精细适配）。

### 5.3 数据安全
- localStorage 仅存储于用户本地设备，不自动上传任何服务器。
- CSV 导出仅在用户主动点击时生成，不自动保存到云端。

## 6. 用户操作流程（关键路径）

### 6.1 每日记账流程
1. 用户打开网页（默认今日日期）。
2. 在“今日总收入”输入框中输入金额（例如 1200）。
3. 页面实时显示“利润 840元”和“分成 360元”。
4. 点击“保存”按钮。
5. 提示“保存成功”，本月/今年统计自动累加，最近记录列表更新。
6. （若当天已存在记录）按钮变为“更新”，用户修改金额后点击更新，旧数据被覆盖，统计重新计算。

### 6.2 补录历史记录
1. 用户点击日期选择器，选择历史日期（例如2026-06-01）。
2. 输入总收入，点击“保存”。
3. 系统检查该日期是否已有记录，若无则新增，若有则提示“是否覆盖？”。
4. 保存后，历史记录页可查看该日数据。

### 6.3 导出数据
1. 用户进入历史记录页。
2. 选择月份（例如2026年6月）。
3. 点击“导出CSV”按钮。
4. 浏览器自动下载文件“收入记录_2026年6月.csv”。
5. 用户可用 Excel 打开查看或打印。

## 7. 产品成功指标（可衡量）

- **用户留存**：次日留存 > 40%（可通过埋点统计，但MVP可不做）。
- **使用频率**：日均记账次数 ≥ 1（目标用户应每天使用）。
- **功能满意度**：通过简单反馈表单（如“是否解决了你的算账痛点？”）收集，目标 > 80% 回答“是”。
- **导出使用率**：≥ 20% 的用户每月至少导出一次数据。

## 8. 风险与缓解措施

| 风险                         | 影响 | 缓解措施                                             |
| ---------------------------- | ---- | ---------------------------------------------------- |
| 用户清除浏览器数据导致记录丢失 | 高   | 在页面添加“导出备份”提示，鼓励用户定期导出CSV；未来提供云同步 |
| 分成比例固定导致部分用户不适用 | 中   | 快速迭代P1功能，允许自定义比例（1-2周内上线）          |
| 移动端体验差                 | 中   | 确保基本响应式，按钮足够大，输入框易点；专门优化放在P1  |
| 用户忘记每天记账             | 低   | 可增加可选的每日浏览器通知提醒（P2）                   |
